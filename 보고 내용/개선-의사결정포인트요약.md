# 경락이(LSB) 개선 — 핵심 의사결정 포인트 요약

작성 2026-06-10 · 요청: 팀 피드백("맥락은 잘 나오는데, 본문의 핵심 의사결정 포인트도 요약해주면 좋겠다")

## 무엇을 바꿨나
기존 답변(맥락·흐름 + 출처)에 더해, 보고·결정성 자료일 때 본문에서 *핵심 의사결정 포인트*를 뽑아 정리하도록 함.

뽑는 항목:
- 무엇을 결정했나
- 사유(왜)
- 누가 · 언제
- 미결 사항(아직 안 정해진 것)

## 적용 범위(조건부)
- 보고·결정·논의성 자료 → 의사결정 포인트 블록 추가
- 단순 "어디 있어요?" 위치 질문, 일반 대화 → 추가 안 함
- 항목이 분명할 때만. 억지로 만들지 않음(없으면 생략).

## 비용·구현
- 추가 비용 0 (벡터DB·임베딩 불필요). 경락이가 이미 읽는 본문에서 뽑는 거라 시스템 프롬프트 규칙만 추가.
- 변경 파일: `src/agent.js` (시스템 프롬프트 [작동 방식] 4번 추가).

## Before / After (요지)
- Before: 맥락·흐름을 줄글로 + 출처
- After: 위 + "핵심 의사결정 포인트"(결정/사유/주체·시점/미결) 블록

## 상태
- 코드 반영 완료, push 후 배포 시 적용. (검토 대기 항목 아님 — 노출/권한과 무관한 표현 개선)
