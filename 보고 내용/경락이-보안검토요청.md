# 경락이(LSB) — 사용·정보보안 정책 검토 요청

작성: 박성훈(경영지원실) · 대상: 검토 승인권자 · 목적: 사내 배포 전 정보 경계/권한 정책 승인

---

## 1. 한 줄 배경

경락이는 경영지원실 Slack(및 추후 Notion/Drive)에 흩어진 지난 보고·결정·문서를 **물어보면 맥락과 출처까지 짚어 찾아주는** 사내 AI 에이전트입니다. 기능은 동작하나, **민감정보 노출 우려가 있어 사용 범위·노출 방식·외부전송 정책을 배포 전에 확정**하고자 합니다.

## 2. 검토가 필요한 이유 (현재 구조의 위험)

- 경락이는 검색 품질을 위해 한 사람의 Slack 검색 토큰(현재 박성훈 계정)으로 **워크스페이스 전체를 검색**합니다.
- 따라서 **누가 질문하든** 그 토큰 소유자가 볼 수 있는 범위(경영진 전용 채널 포함)까지 끌어올 수 있고, **질문자 본인의 열람 권한은 반영되지 않습니다.**

### 구체적 우려
1. **경영진 전용 채널 내용**이 일반 사용자의 질문에 딸려 나올 수 있음.
2. 대표가 특정 임원 채널에 속해 있지 않은데도, 경락이가 **그 임원 관련 내용**을 끌어올 수 있음.

## 3. 검토·결정이 필요한 항목 (체크리스트)

| # | 항목 | 선택지 | 경영지원실 제안 |
|---|---|---|---|
| 1 | **사용 권한 범위** | 전원 / 특정 직책 / 특정 인원 | **우선 대표 1인 전용**으로 시작 |
| 2 | **답변 노출 방식** | 채널 공개 / 본인만(비공개) / DM 전용 | **DM 전용**(채널에 내용 노출 안 함) |
| 3 | **외부전송 정책** | 허용 / 범위 제한 / 불가 | 답변 생성에 외부 LLM(Anthropic API) 전송이 **전제됨** → 허용 범위 확정 필요 |
| 4 | **검색 대상 범위** | 전체 / 지정 채널만 / 민감채널 제외 | 민감 채널 포함 여부 결정 필요 |
| 5 | **로깅·감사** | 질의응답 기록 보관 여부 / PII 처리 | PII 마스킹 적용, 보관 정책 결정 필요 |
| 6 | **장기 권한 모델** | 사용자별 열람권한(ACL) 도입 여부 | 2단계 과제로 검토 |

## 4. 경영지원실 제안안 (초안)

**"대표 1인 전용 + DM 전용 응답"** 으로 시작.

- 지정된 사용자(대표)만 경락이를 호출 가능.
- 채널에서 부르더라도 **답변은 채널이 아니라 호출자 DM으로만** 전달 → 옆사람에게 내용이 노출되는 사고 방지.

### 이 제안의 효과와 한계 (중요)
- **효과**: 답변이 공개 채널에 노출되는 사고를 차단. 사용 인원을 최소화해 위험 면적 축소.
- **한계**: DM 전용은 "누가 답을 **보느냐**"를 막는 것이지, "질문자가 그 내용을 **볼 자격이 있느냐**"를 막는 건 아님. 즉 대표 본인은 검색된 내용을 그대로 보게 됨.
  → "질문자 본인도 권한 밖 내용은 못 보게" 하려면 **사용자별 열람권한(ACL, 항목 6)** 이 추가로 필요하며, 이는 별도 개발 단계입니다.

## 5. 현재 상태

- **라이브(배포됨)**: 채널 응답, 지정 채널 검색, 전체기간 검색, 출처 표기, 멘션 이름 복원 등 기능 동작.
- **미반영(승인 대기)**: 항목 1·2(대표 전용 + DM 전용)는 **코드 작성만 해두고 배포하지 않음**. 승인 후 적용 예정.
- **미확정(관문)**: 항목 3(외부전송 정책). 인계 단계부터 폴백 담당 경묵 확인 필요로 기재돼 있던 사안.

## 6. 요청 사항

위 3번 표의 항목들, 특히 **① 사용 권한 범위 ② 답변 노출 방식 ③ 외부전송 정책**에 대한 검토·승인을 요청드립니다. 승인되는 방향에 맞춰 적용하겠습니다.

## (부록) Claude Code 인계 메모 — 승인 후 구현 대기

- **미배포 코드**: "대표 전용 + DM 전용" 버전 `index.js`가 작성돼 있었으나, 미승인 상태라 현재는 채널 응답 버전으로 되돌려 둠. 승인 시 아래를 적용:
  - `LSB_ALLOWED_USERS` 에 대표 멤버 ID 지정(환경변수).
  - `index.js` 응답 경로를 DM 전용으로 전환(`conversations.open` → DM 채널에 응답, 채널엔 ephemeral 안내만). **Slack `im:write` 스코프 추가 + 재설치 필요.**
- **ACL(항목 6) 구현 시**: 질문자 멤버십 기준 필터 or 사용자별 토큰 기반. 색인에 `permission_tag` 메타데이터는 이미 부착돼 있어 토대는 마련됨.
- 전체 구조·실행·로드맵은 동봉 `HANDOFF.md` / `LSB-구축계획서.md` 참조.
