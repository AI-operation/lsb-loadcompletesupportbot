# 경락이 2.0 — 최종 고도화 로드맵 (확정안)

작성 2026-06-11 · 오너 박성훈(경영지원실) · 이 문서가 최종 방향이며, 이후 작업은 여기 단계 순서를 따른다.

## 0. 결론 한 줄
경락이를 "물어보면 찾아주는 검색봇"에서 "경영지원실의 기억·실행을 책임지는 운영 두뇌"로 올린다. 토대는 하나(구조화 색인), 그 위에 3대 기능을 순서대로 얹는다. 더 늘리지 않는다.

## 1. 키스톤 — 구조화 색인 (모든 것의 토대)
대상 채널의 모든 메시지·스레드·문서를 한 번 색인하고 이후 증분 유지한다. 각 항목에 메타데이터를 붙인다: 작성자 / 멘션된 사람 / 채널 / 시각 / 스레드 / 링크·첨부 / 본문에서 추출한 날짜·기한 / 임베딩(의미벡터).
- 이게 있어야 "한 사람 언급 전부", "무거운 질문에도 안 터짐", "의미로 찾기", "기한 추적", "결정 계보"가 전부 가능해진다.
- 지금 약점(태그 일부만 보임 / 대량 질문 실패)의 근본 해결책.

## 2. 3대 기능 (우선순위 고정, 이외 추가 안 함)
1. 완전·의미 검색 — 색인 기반으로 빠짐없이(exhaustive) + 뜻으로(semantic) 찾는다. 사람·기간·주제 조합 필터. (현재 검색을 대체)
2. 기한·갱신 레이더 — 색인에서 날짜 붙은 것(계약 만료, 보험·라이선스 갱신, 정기 정산일, 법정교육, 수습 리뷰 등)을 모아 매일/매주 DM으로 "곧 마감/만료" 선제 알림. 묻기 전에 가져다준다.
3. 정책 계보 + 결정 원장 — 결정이 날 때마다 무엇을/왜/누가/언제/근거를 구조화해 쌓고, 현재 규정의 변천사를 근거째 역추적("왜 이렇게 됐어?"에 영수증으로 답).

## 3. 단계 (이 순서대로만 진행)
- Phase 0 (지금): 그동안 만든 개선(전체기간 검색·멘션 복원·서식·의사결정 포인트·강력모드·정보범위 완화)을 push & 배포, 오너 단독 테스트.
- Phase 1 (토대): 구조화 색인 구축 — DB + 임베딩 + 1회 백필 + 증분 동기화 → 검색을 완전·의미 검색으로 교체. [3대 기능 #1 완성, 태그누락·대량실패 근본 해결]
- Phase 2 (선제): 기한·갱신 레이더 — 날짜 추출 + 예약 발송(아침/주간 DM 브리핑). [3대 기능 #2]
- Phase 3 (기억): 정책 계보·결정 원장 — 결정 이벤트 구조화 축적 + 변천사 역추적. [3대 기능 #3]

## 4. 확정 기술 스택
- 색인 DB: Render PostgreSQL + pgvector (기존 Render 스택 안에서 해결, 별도 벤더 없음)
- 임베딩: Google Gemini (gemini-embedding-001) — GWS 생태계와 동일, 다국어·한국어 우수, $0.15/1M(배치 시 절반). [대안: Voyage]
- 적재: 최초 백필 1회 + 새 메시지 이벤트로 증분 + 야간 스케줄로 정합성 보정
- 선제 알림: Render 예약 작업(cron) → DM 발송
- 답변 모델: claude-sonnet-4-6 (유지)
- 봇 형태: 현행 HTTP/Events(Render Web Service) 유지

## 5. 준비물 (오너가 제공해야 진행 가능)
- Gemini API 키 (ai.google.dev 발급) — 또는 GCP Vertex AI 사용
- Render PostgreSQL 인스턴스(소액/무료 티어)
- (Phase 2 이후) 외부전송·정보경계 정책 사인오프 — 보안 검토 결과

## 6. 비용 (현실)
- 1회성: 채널 전체 임베딩 백필 = 소액(수천 원 수준)
- 상시: Postgres 소액 + Anthropic 답변 토큰(지금도 발생 중). 선제 브리핑은 호출이 늘어 약간 추가.

## 7. 보안 트랙 (위 로드맵과 병행·독립)
- 대표 전용 / DM 전용 응답, 사용자별 열람권한(ACL)은 별도 검토 안건(`경락이-보안검토요청.md`)으로 진행. 승인 결과에 맞춰 적용하며 위 Phase와 충돌하지 않는다.

## 8. 성공 기준 (이렇게 되면 성공)
- "실장님 언급 전부" → 빠짐없이 나온다.
- "작년 ○○ 전부 정리" → 안 터지고, 많으면 나눠서라도 끝까지 답한다.
- 아무도 안 물어도 마감·갱신을 먼저 알려준다.
- "이 규정 왜 이렇게 됐어?" → 변천사+근거로 답한다.

## 9. 1순위 착수 지점
Phase 1(구조화 색인). 이거 하나가 3대 기능의 토대이자 현재 약점의 근본 해결이다. 준비물(Gemini 키 + Postgres)만 갖춰지면 바로 착수 가능.
