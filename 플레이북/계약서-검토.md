# 플레이북 · 계약서 검토

> 경락이가 계약서 검토 요청을 받으면 이 절차를 따라 **항목별로 점검**하고,
> 사람이 최종 확인하도록 결과를 정리한다.
>
> ⚠️ 이 문서는 검토를 돕는 체크리스트이지 **법률 자문이 아니다.**
> 구체적 법령·금액·기한은 항상 **최신 법령과 사내 규정으로 대조**하고,
> 중요한 건은 책임자·전문가 검토를 받는다. 경락이는 "판정"하지 않고 "확인할 것을 표시"한다.

---

## 1. 적용 시점
신규 계약 체결 전, 갱신·변경 전, 외부(상대방 양식)에서 받은 계약서 검토 시.

## 2. 기본 정보 점검
- **당사자**: 법인명·대표자·사업자번호가 우리 등기/상대방 등기와 일치하는가
- **계약 목적·범위**: 실제 합의와 문구가 일치하는가, 모호하거나 빠진 범위 없는가
- **기간**: 시작·종료일, 자동갱신 여부, 갱신 거절 통지 기한(언제까지 통지해야 하는지)
- **대금**: 금액 / 부가세 포함 여부 / 지급 시기·조건 / 지연이자 / 정산 방식
- **서명**: 권한 있는 자의 서명·날인, 직인/계인, 페이지 누락·간인 여부

## 3. 위험 조항(독소조항) 점검
- 상대방만 갖는 **일방적 해지권**, 우리에게만 불리한 **위약금/손해배상**
- **책임 한도 부재**(무제한 배상), 면책 범위가 과도
- **지식재산권·산출물 귀속**이 우리에게 불리
- **비밀유지·경업금지**의 범위·기간이 과도하거나 일방적
- **준거법·관할**이 상대방에 유리(원격지·해외 법원)
- **자동갱신 + 해지통지 기한**이 비대칭(우리는 길고 상대는 짧음)
- **하도급·재위탁·권리양도**를 상대방이 무단으로 할 수 있는가
- **개인정보·보안** 처리 조항이 법령(개인정보보호법 등) 요건을 충족하는가  ⚠️확인필요

## 4. 우리 표준과의 대조
- 사내 **표준계약서/표준조항**과 다른 부분을 모두 표시
- **승인 한도**(금액·기간) 초과 여부 → 해당하면 결재 라인 확인  ⚠️확인필요
- 과거 유사 계약과 조건 차이(불리해진 부분)

## 5. 산출물 형식 (경락이가 답하는 방식)
항목마다 다음 중 하나로 표시한다:
- ✅ **이상 없음**
- ⚠️ **확인 필요** — 사유 + 무엇을 확인해야 하는지
- ❌ **누락/위험** — 사유 + 권고

그리고 마지막에:
1. **핵심 위험 Top 3** 요약
2. **사람이 반드시 확인할 사항** 목록
3. **근거** — 계약서의 해당 조항 위치를 인용
4. 마무리 한 줄: _"이는 검토 보조이며, 최종 법적 판단은 담당자·전문가 확인이 필요합니다."_

(작성/수정: 경영지원실 · 최종 검토: 책임자)
